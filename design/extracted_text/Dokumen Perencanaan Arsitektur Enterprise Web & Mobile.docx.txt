2. Catatan Penting tentang “Chrome App”
Untuk kebutuhan “Chrome App”, pilihan enterprise yang benar pada 2026 adalah PWA installable, bukan Chrome Apps klasik. Google menyatakan Chrome Apps mulai dihentikan bertahap sejak Juli 2025 dan mencapai end-of-life pada Oktober 2028; jalur migrasi yang direkomendasikan adalah web applications atau Chrome Extensions, dan web applications adalah jalur utama untuk mayoritas kasus.
Maka keputusan arsitektur:
Android     : React Native AppiOS         : React Native AppChrome App  : Next.js PWA yang bisa di-install dari ChromeWeb Desktop : Next.js Responsive Web AppAdmin Web   : React Dashboard
3. Mengapa Next.js Dipilih untuk Frontend Pengguna Umum Web?
Next.js dipilih dibanding Nuxt/Vue untuk aplikasi pengguna umum karena kebutuhan Anda menekankan SEO, performa, realtime, enterprise maintainability, dan mobile/PWA. Next.js App Router mendukung fitur React modern seperti Server Components, Suspense, dan Server Functions, sedangkan Metadata API dapat digunakan untuk SEO dan web shareability.

4. Mengapa React Dipilih untuk Frontend Admin Web?
Untuk admin dashboard, SEO tidak terlalu penting. Yang paling penting adalah:
Realtime dataData table besarFilter kompleksRole-based UIGrafik monitoringApproval workflowNotification centerAudit logExport Excel/PDF
Maka admin web sebaiknya dibuat sebagai React SPA/dashboard dengan TypeScript, Tailwind, TanStack Query, WebSocket client, dan charting library.
Keputusan
Frontend Admin Web:React + TypeScript + Tailwind CSS + Dashboard Layout
Catatan
Untuk admin dashboard, tidak wajib menggunakan Next.js. React + Vite lebih ringan dan cepat untuk dashboard internal. Namun jika ingin semua web berada dalam satu framework, admin juga boleh dibuat dengan Next.js.

8. Database
Keputusan
PostgreSQL
PostgreSQL cocok untuk aplikasi enterprise karena kuat untuk data relasional, transaksi, integritas data, indexing, JSONB, full-text search, dan ekstensi. PostgreSQL juga dikenal memiliki reputasi kuat untuk reliability, data integrity, extensibility, dan ACID compliance.
Pola Database Microservices
Gunakan prinsip:
Database per service, bukan satu database besar untuk semua service.
Contoh:
user_service_dbbooking_service_dbpayment_service_dbnotification_service_dbaudit_service_dbreporting_service_db
Untuk integrasi data antar service, gunakan event broker, bukan join langsung lintas database.
9. Cache dan Realtime Layer
Redis
Redis digunakan untuk:
Cache APISession cacheRate limitingTemporary tokenOTP cacheRealtime presenceDistributed lock ringanQueue ringan non-kritis
Redis sangat baik untuk cache dan data cepat. Namun Redis Pub/Sub memiliki sifat at-most-once delivery, artinya pesan bisa hilang jika subscriber gagal menerima pesan. Karena itu, Redis Pub/Sub tidak boleh menjadi satu-satunya mekanisme untuk event bisnis penting.
Event Broker Enterprise
Untuk microservices yang benar-benar enterprise, tambahkan:
NATS JetStream
NATS JetStream menyediakan persistence, message replay, dan konfigurasi high availability; dokumentasi NATS menyebut JetStream dapat menyimpan pesan dan memutar ulang pesan ke consumer saat diperlukan.
Pola Realtime
Browser/Mobile    ↓ WebSocket / SSERealtime Gateway    ↓NATS JetStream    ↓Microservices    ↓PostgreSQL + Redis
Kapan Pakai WebSocket?
ChatLive statusLive trackingNotifikasi realtimeDashboard monitoringPresence online/offline
Kapan Pakai Server-Sent Events?

10. Authentication dan Authorization
Keputusan
Keycloak + OpenID Connect/OAuth2 + JWT
Keycloak dipilih karena mendukung OpenID Connect, OAuth2, SAML, Single Sign-On, dan authorization services.
Kenapa Bukan JWT Manual Saja?
JWT manual cukup untuk aplikasi kecil, tetapi kurang ideal untuk enterprise karena perlu membangun sendiri:
SSORefresh token rotationRole managementClient managementSession revocationMulti-tenant realmIdentity federationAudit loginPassword policyMFAOAuth2/OIDC compliance

23. Prinsip Penutup
Arsitektur ini tidak hanya cocok untuk aplikasi Puskeswan, tetapi juga untuk aplikasi enterprise lain seperti portal layanan, LMS, sistem antrian, dashboard pemerintahan, marketplace layanan, aplikasi booking, sistem pengaduan, manajemen dokumen, dan aplikasi mobile layanan publik.
Keputusan paling penting adalah:
Gunakan Next.js untuk public web karena SEO dan PWA.Gunakan React untuk admin dashboard karena realtime dan fleksibilitas.Gunakan React Native untuk Android/iOS karena satu ekosistem dengan React.Gunakan Golang untuk backend karena cepat dan scalable.Gunakan PostgreSQL untuk data utama karena relasional dan reliable.Gunakan Redis untuk cache, bukan event bisnis kritis.Gunakan NATS JetStream untuk event-driven microservices.Gunakan Keycloak untuk enterprise SSO.Gunakan PWA sebagai pengganti Chrome App.
